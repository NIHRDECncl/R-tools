Welcome to the NIHR DEC Newcastle's tool for exploring:

  1. How results from a diagnostic test help with clinical decision-making.
    • Result = rule in warrants treat the condition
    • Result = rule out warrants do not treat the condition
    • Result = inconclusive may warrant further investigation
  2. How the results from (and the decisions warranted by) a test depend on:  
    • Pre-test probability (i.e. the prevalence of the condition)
    • Sensitivity of the test (i.e. the true poitive rate)
    • Specificity of the test (i.e. the true negative rate)
  3. How decisions depend on the trade-off between benefits and harms, and how this determines the threshold post-test probability at which "rule in" or "rule out" decisions should be made.   
This is the second of three ShinyApps for exploring the factors that determine a diagnostic test's accuracy:
  1. Test accuracy - sensitivity, specificity, true and false postives, true and false negatives
  2. Clinical accuracy and clinical utility  - positive and negative predictive values; probability thresholds for rule-in and rule-out decisions
  3. Uncertainties in assessing test accuracy when the reference standard is not perfect.

Use the left-hand panel to (re)draw the graphs and enter the variables

  • A button at the top to "Update the graphs", which are shown under the tabs "Decision thresholds" and "Pre- to post-test probablity".
  • The "Input variables" which are used to:
    • label the graphs
    • set the variables which determine:
      • the post test probability: disease prevalence and test accuracy (sensitivity and specificity)
      • the decision-making: thresholds for rule-in results and rule-out results
      • the precision of post-test probability estimations: the population size in the study which provides the sensitivity and specificity estimations

 Use the tabs to review information about using the App and interpreting the results, and to view the graphs
