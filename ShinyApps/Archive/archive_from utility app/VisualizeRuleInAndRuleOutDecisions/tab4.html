Interpreting graph 2:Assessing the clinical utility of a diagnostic test 

Graph 2 illustrates how decisions are made with information from a diagnostic test. The clinical utility of the test is the net benefit to patients, and is determined by the outcomes which follow from the management decisions made on the basis of the test's results. Thus, the first step to assessing clinical utility is to understand the factors that determine decision-making.    

The graph on the Decisions tab is dynamic: it allows you to study a specific condition, test to diagnose the condition, and the clinical decisions that can be made on the basis of the test result:.

The graph allows you to explore 

  1. the probability of a disease or condition before a diagnostic test (i.e. the prevalance of the condition), and
     
  2. the probabilities after postive and negative test results (i.e. the post test probabilities: PPV (postive predictive value) and NPV (negative predictive value)).

  3. The thresholds which post test probabilities must pass for a test result to support a clinical decision.

The graph shows a lot of information. But, if you want to understand the value and and appropriate use of diganostic tests, it is worth investing some effort learning to read the graph. Each element of the graph is explained below with reference to the (static) scree snip in the figure below.

Although the graph might seem to be a normal X-Y plot, what looks like an X-axis is a visual metaphor showing how test results move from pre-test probability on the left to post-test probability on the right.

The decision threshold for a positive test (line 3)
and
The post-test probability for a positive test (PPV) (line 7)

If a test is positive, the result may be used to justify clinical decisions, such as "rule in the disease and manage accordingly".
Note that a positive result may only be used to justify a rule in decision if the post-positive test probability is above the rule in decision threshold.

You can change the specificity and prevalence settings on the left to see how this changes the post-positive test probability (i.e. positive predictive value, PPV), and check if it exceeds the decision threshhold.  

Change the sensitivty settings on the left and check that this does not changes the post-positive test probability.

Change the population size and see how the precision (95% confidence interval) changes --- this is shown as the salmon pink ribbon around the post-positive test line.
If the confidence interval includes the decision threshold, you should not be confident in using the result to justify a rule in decision.

The decision threshold for a negative test (line 4)
and
The post-test probability for a negative test (NPV) (line 6)

Similarly, if a test is negative, the result may be used to justify clinical decisions, such as "rule out the disease and manage accordingly".
Note that a negative result may only be used to justify a rule out decision if the post-negative test probability (i.e. negative predictive value, NPV) is below the rule out decision threshold.

You can change the sensitivity setting on the left to see how this changes the post-negative test probability, and check if it falls below the decision threshhold.  

Change the specificity settings on the left and check that this does not changes the post-negative test probability.

Change the population size and see how the precision (95% confidence interval) changes --- this is shown as the sea green ribbon around the post-negative test line.
If the confidence interval includes the decision threshold, you should not be confident in using the result to justify a rule out decision.

The pre-test probability (prevalence) (line 5)

The pre-test probability of the condition, its prevalence, is shown by the horizontal green line (numbered 5).

You can change the prevalence setting on the left to see how this changes the post test probabilities. See at what levels the prevalence must be for the post test probabilities to just fail to cross the decision thresholds. The significance of this is that, if the post test probability with a positive result is less than the decision threshold, there is no point in doing the test to make the diagnosis. Similarly, if the post test probability with a negative result is greater than the decision threshold, there is no point in doing the test to rule the diagnosis out.   

Where do I find the data?

Data for your condition's prevalence and the test's accuracy (sensitivity and specificity) will be in the scientific literature, unless the test is under development, in which case unpublished data from the developer may be available.Use data from a (well conducted and well reported) systematic review and meta-analysis, if this is available. Otherwise, use data from a (well conducted and well reported) study. Data for the population size should be available from the the source of your accuracy data. 

Obtaining reliable data for the decision thresholds can be a challenge. But it is a challenge worth rising to, as the decision thresholds determine the clinical utility of the test.

Health economists do the kind of studies that determine the decision threshold that provide the optimum cost for net benefit --- net benefit takes account of benefits and the harms. Costs, benefits, and harms are those that follow from doing the test and acting on its result.

In technical terms, the recipe that the health economists folllow is:

Decision threshold = (net harm)/(net harm + net benefit)
Net harm = utility(no Rx | no disease) - utility(treatment | no disease)
Net benefit = utility(Rx | disease) - utility(no treatment | disease)ease)

Because this type of health economic study is difficult and expensive, clinicians often have to use intuition and gut feeling to decide on decision thresholds. 

Informal setting of decision thresholds can be biased, either by over-emphasising the benefits in comparison with the harms, or vice versa.
